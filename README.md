# Путешествие героини · Роза из «Титаника»

Лонгрид-разбор пути Розы по схеме Морин Мёрдок «Путешествие героини».
Сделано на React + Vite + Tailwind, контент хранится в одном JSON-файле.

---

## Структура уроков (этапов)

Весь контент путешествия лежит в одном файле:

- **`src/data/rose.json`** — единственный источник правды для всех этапов и эпилога.

Формат:

```json
{
  "stages": [
    {
      "title": "1. Название этапа",
      "body": "Абзац 1.\nАбзац 2.\nАбзац 3. ...",
      "images": ["", "url-ко-2-му-абзацу", "", ...]
    }
  ],
  "conclusion": "Текст эпилога..."
}
```

Правила:

- `stages` — массив из 10 этапов пути героини.
- `title` — заголовок этапа в формате `"N. Название"` (порядковый номер + точка обрезаются при выводе).
- `body` — текст этапа. **Каждый абзац разделяется переводом строки `\n`.**
  Разбиение на абзацы выполняется в `src/pages/Stage.tsx`:
  ```ts
  const paragraphs = stage.body.split("\n").filter((p) => p.trim());
  ```
- `images` — массив URL иллюстраций, **индекс соответствует индексу абзаца**:
  - `images[0]` — иллюстрация к 1-му абзацу,
  - `images[1]` — ко 2-му и т.д.
  - Пустая строка `""` означает «без иллюстрации».
  - Массив можно делать короче количества абзацев — лишние абзацы просто будут без картинки.
- `conclusion` — текст финального экрана (страница `/conclusion`).

Каждый абзац рендерится компонентом `src/components/ParagraphBlock.tsx`,
который опционально показывает картинку из `images[i]` рядом с текстом.

## Маршруты

Маршрутизация описана в `src/App.tsx` через `react-router-dom`
(используется `BrowserRouter` — Lovable-хостинг сам делает SPA-fallback).

Все основные страницы рендерятся внутри общего `Layout` (с боковой навигацией `SideNav`):

| Путь            | Компонент           | Назначение                                              |
|-----------------|---------------------|---------------------------------------------------------|
| `/`             | `pages/Home.tsx`    | Титульная страница: введение, список этапов             |
| `/stage/:id`    | `pages/Stage.tsx`   | Один этап пути. `:id` — число от 1 до 10                |
| `/conclusion`   | `pages/ConclusionPage.tsx` | Эпилог · Архетип (текст из `conclusion`)         |
| `*`             | `pages/NotFound.tsx`| 404 (вне `Layout`)                                      |

Поведение `/stage/:id`:

- `id` парсится в число; если он вне диапазона `1..stages.length`, происходит `Navigate` на `/`.
- При смене этапа страница скроллится в начало и заново проигрывается анимация появления.
- Внизу страницы — `StagePager` с переходом к предыдущему/следующему этапу.

Боковая навигация `src/components/SideNav.tsx`:

- Ссылка на `/` (Введение).
- Список из 10 ссылок на `/stage/1` … `/stage/10` (заголовки берутся из `rose.json`).
- Ссылка на `/conclusion` (Эпилог).
- На мобильных открывается по кнопке-гамбургеру.

## Публикация

Проект публикуется через Lovable.

**Опубликовать / обновить:**

- **Десктоп:** кнопка **Publish** в правом верхнем углу редактора.
- **Мобильный:** в режиме Preview — кнопка **…** в правом нижнем углу → **Publish**.

После первой публикации проект доступен по адресу вида
`https://<имя>.lovable.app` (текущий: `https://rose-heroine-path.lovable.app`).

**Что когда уезжает в прод:**

- Изменения фронтенда (текст в `rose.json`, компоненты, стили) — попадают в прод
  только после нажатия **Update** в диалоге публикации.
- Бэкенд-изменения (если появятся edge-функции/миграции) — деплоятся автоматически.

**Свой домен:** доступен после первой публикации в
**Project Settings → Domains** или прямо из диалога Publish → **Add custom domain**.

**Превью-ссылка** (`id-preview--*.lovable.app`) по умолчанию требует логина в Lovable.
Чтобы поделиться без логина — **Share → Share preview** (ссылка живёт 7 дней).

**Глубокие ссылки и обновление страницы** работают из коробки: Lovable-хостинг
делает SPA-fallback на `index.html`, никаких `_redirects`/`vercel.json` не нужно.

## Как добавить иллюстрацию к абзацу

1. Открыть `src/data/rose.json`.
2. Найти нужный этап в массиве `stages` (индекс = номер этапа − 1).
3. В его `images` поставить URL картинки на позицию `номер_абзаца − 1`.
   Если массив короче — дополнить пустыми строками `""` до нужной позиции.

Пример — иллюстрация ко 2-му абзацу 3-го этапа:

```json
"images": ["", "https://example.com/pic.jpg"]
```
